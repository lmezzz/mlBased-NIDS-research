# mlBased-NIDS-research

> Cross-dataset generalization study of ML-based Intrusion Detection Systems
> with protocol-level analysis of dataset shift.
> **Course:** Network Protocols & Security (NPS)

---

## Paper Title

**"Dataset Shift in ML-Based Intrusion Detection Systems: A Protocol-Level Analysis of Generalization Failure"**

---

## Research Question

> "Do ML models trained on NSL-KDD generalize to CICIDS 2017, and can the performance gap be explained at the network protocol level?"

## Experiment Design

### Feature Sets

**EXP0 — Control (Protocol-Unaware)**
Features that require no protocol grammar knowledge:
```
duration, src_bytes, dst_bytes, bytes_per_sec, byte_ratio,
protocol_tcp, protocol_udp, protocol_icmp
```

**EXP1 — Swap src_bytes → syn_ratio**
```
duration, syn_ratio, dst_bytes, bytes_per_sec, byte_ratio, protocol_*
```

**EXP2 — Swap dst_bytes → rst_ratio**
```
duration, syn_ratio, rst_ratio, bytes_per_sec, byte_ratio, protocol_*
```

**EXP3 — Add fin_ratio**
```
EXP2 features + fin_ratio
```

**EXP4 — Add data_pkt_ratio**
```
EXP3 features + data_pkt_ratio
```

**EXP5 — Add service_bucket**
```
EXP4 features + svc_http, svc_ssh, svc_ftp, svc_dns, ...
```

### Protocol-Aware Feature Mapping

| Feature | KDD Source | CICIDS Source | Protocol Meaning |
|---|---|---|---|
| `syn_ratio` | `serror_rate` | `SYN Flag Count / Total Fwd Packets` | SYN flood intensity |
| `rst_ratio` | `rerror_rate` | `RST Flag Count / Total Fwd Packets` | Port scan / closed port hits |
| `fin_ratio` | `flag == "SF"` | `FIN Flag Count / Total Fwd Packets` | Connection lifecycle completion |
| `data_pkt_ratio` | `src_bytes > 0` | `act_data_pkt_fwd / Total Fwd Packets` | Payload presence |
| `service_bucket` | `map_service_to_bucket(service)` | `port_bucket(Destination Port)` | IANA protocol tier |

### Complete Results Table — All Models All Experiments

| EXP0 | Control | 0.7363 | 0.2312 | 0.5051 | 0.7991 | 0.2621 | 0.5370 | 0.7362 | **0.3011** | **0.4351** |
| EXP1 | +syn_ratio | 0.7387 | 0.2428 | 0.4959 | 0.7694 | 0.2617 | 0.5076 | 0.7473 | 0.2312 | 0.5161 |

| EXP5 | +service | 0.7466 | 0.1938 | 0.5527 | **0.8247** | 0.1519 | 0.6728 | 0.7617 | 0.2198 | 0.5419 |

### Key Findings

**Finding 1 — Protocol features consistently worsen generalization**
Every protocol-aware feature added after EXP0 degrades cross-dataset F1.
The best cross-dataset performance across all experiments is SVM EXP0 (0.3011) — the control baseline.

**Finding 2 — rst_ratio is the most damaging single feature**
Adding rst_ratio (EXP2) causes the largest single-experiment drop across all three models.
RST-based attacks (port scanning) largely disappeared from CICIDS 2017 — modern attacks use HTTP floods that generate no RST traffic.

**Finding 3 — The SVM paradox**
SVM achieves the best cross-dataset F1 without protocol features (0.3011) but collapses to 0.0201 when rst_ratio is added — the worst result of any model-experiment combination.
Maximum margin commits strongly to a decision boundary that becomes directionally reversed in CICIDS.

**Finding 4 — syn_ratio is the only protocol feature that doesn't hurt**
Adding syn_ratio (EXP1) causes minimal damage and slightly improves LR cross-dataset F1.
syn_ratio is near zero for both normal and attack traffic in CICIDS — adds minimal noise in either direction.

**Finding 5 — LR EXP4 is the unexpected best performer**
LR with data_pkt_ratio achieves 0.3105 cross-dataset F1 — highest of any experiment.
data_pkt_ratio captures payload presence, which partially distinguishes Slowloris-style attacks in CICIDS.

**Finding 6 — Within-dataset F1 is a misleading metric**
RF EXP5 achieves highest within-dataset F1 (0.8247) but poor cross-dataset F1 (0.1519).
The model that looks best on the benchmark is consistently not the best real-world performer.

## Core Argument

Protocol-aware ML features improve within-dataset IDS performance but worsen cross-dataset generalization. The 14-year gap between NSL-KDD (2003) and CICIDS 2017 represents not just statistical drift but a fundamental migration of attack behavior from Layer 3/4 protocol grammar violations to Layer 7 application semantics. Features grounded in attack techniques (SYN floods, RST-based scanning) fail catastrophically when those techniques evolve. Features grounded in RFC-defined protocol rules show marginally more stability. No protocol-aware feature combination outperforms the simple control baseline on cross-dataset F1.
